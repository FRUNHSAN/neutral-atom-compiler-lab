| 离子阱 | 小 | ⭐最高 | 慢 | s | 电磁笼里的糖葫芦 |

## 一、物理层：比特长什么样

### 1.1 什么是中性原子比特

**中性原子** = 不带电的原子（通常是铷 Rb-87 或锶 Sr-88），在超高真空中被激光冷却到 μK 级温度，然后用聚焦激光"夹"住。

每个原子是一个天然的量子比特——原子的两个超精细基态能级编码 |0⟩ 和 |1⟩。所有同种原子完全全同（不像超导比特需要逐个校准），天生适合规模化。

**相干时间**：秒级（超导 ~ μs，离子阱 ~ s）。寿命长 → 可以做更多操作 → 更适合大规模算法。

> [primer/03]

### 1.2 量子比特的四种基本操作

```
单比特门     用微波/拉曼激光旋转单个原子的量子态     保真度 ~0.9997
双比特门     纠缠两个原子（CZ 门）                   保真度 ~0.995
测量         用荧光成像读原子的态（亮=1, 暗=0）      保真度 ~0.999
空闲等待     啥都不做——但退相干一直在进行             损失来自 T2 衰减
```

**关键事实**：双比特门的保真度（~99.5%）远低于单比特门（~99.97%）。编译器优化的大部分工作就是在减少双比特门数量或减少双比特门操作期间的损耗。单比特门保真度太高了——在 10-30 比特尺度上，单比特门损失几乎可以忽略，这也是为什么 ZAP 的 fidelity breakdown 中 1Q gate bar 总是最窄的。

> [gloss/03-metrics], [ZAP Table I]

### 1.3 退相干：量子计算的终极敌人

**一句人话**：量子态是个"怕被看"的状态。环境的被动偷看（热噪声、杂散电磁场、材料缺陷）会让量子态在未被主动测量的情况下崩溃——这就是退相干。量子计算的全部工程努力都在对抗它。

两个关键时间尺度：
- **T1（能量弛豫时间）**：原子从 |1⟩ 自发掉回 |0⟩ 的平均时间
- **T2（相干时间）**：量子相位信息保持的时间——总比 T1 短。（ZAP 默认 T2 = 1.5×10⁶ μs = 1.5 秒）

ZAP 的 fidelity 公式中，T2 控制退相干通道：F_coherence = exp(-t_idle / T2)。原子每多闲置 1 微秒，保真度就掉一点。这是编译器无法消除的固定税——只能通过减少闲置时间来降低。

## 二、力层：怎么操控原子

### 2.1 光镊系统：SLM + AOD

中性原子路线的"手"和"底盘"，2018 年诺贝尔物理学奖技术。

```
SLM（空间光调制器）          AOD（声光偏转器）
"光的乐高底板"               "光的机械臂"
一次性铺出上千个静态光阱      高速移动少数动态光阱
生成存储阵列                 搬运原子
间距 ~6 μm                  速度 ~μs/μm
```

**静动分工**：SLM 全息生成大规模静态阱（存储区），AOD 用射频频率合成同时驱动多个动态阱做并行搬运（搬原子进出纠缠区）。Caltech 2025 年已铺 ~12,000 个阱位、装载 6,100 个原子。

**这是中性原子最大的路线级优势**——原子可以搬。超导比特焊死在芯片上，想纠缠两个不相邻的比特必须插 SWAP 门链（一堆额外操作）。中性原子直接把两个原子搬到一起纠缠——省了 SWAP 链，但付出了搬运本身的代价（时间 + 原子丢失风险）。

> [gloss/04-materials-devices]

### 2.2 里德堡阻塞：怎么做双比特门

**一句人话**：把一个原子的最外层电子激发到极高能级（里德堡态），原子体积膨胀数千倍，变成"巨人"。巨人周围一定半径内（阻塞半径 ~4.3 μm），强范德华力使邻居无法被同时激发——"一人称王，邻座禁言。"

这是中性原子路线的**双比特门引擎**。中性原子不带电、平时几乎互不理睬（这是好事：串扰小），要纠缠时临时"充值"成里德堡巨人强行发生相互作用——一开一关之间，兼得了隔离性与相互作用。

CZ 门协议：激光脉冲序列利用"阻塞时能级移动"给量子态附加条件相位。阻塞半径可以调整——选主量子数 n（作用强度 ∝ n¹¹）和原子间距，让"该纠缠的在半径内、不该的在半径外"。

**注意**：阻塞不是硬墙——它是 van der Waals 势 ~1/R⁶ 的概率梯度。间距 4.0 μm 和 4.6 μm 处同时激发的概率不是 0 和 1，是 10⁻³ 和 10⁻¹。ZAP 把阻塞当硬墙（"间距 4 μm < 4.3 μm → 不串扰"）是工程近似——间距实际上是一个可调的编译参数。

> [gloss/01-phenomena], [ZAP §II-A]

### 2.3 范德华力 (van der Waals)：阻塞的力学来源

非共振里德堡态之间的长程相互作用：C₆/R⁶。C₆ ∝ n¹¹（主量子数的 11 次方）——对 n 极端敏感。这就是"临时充值变巨人"能奏效的原因：平时 n 低，C₆ ≈ 0（互不理睬）；激发到 n~70-100，C₆ 暴涨十几数量级。

**距离即开关**：R⁶ 的陡峭标度使"半径内极强、半径外近零"成为干净的数字开关——阻塞半径由此可精确设计。

> [gloss/01-phenomena]

### 2.4 绝热定理：原子搬运的物理保障

如果你足够缓慢地改变量子系统的环境，系统会一直乖乖待在基态里跟着变——"温水煮青蛙"式的状态搬运。

这是原子搬运可行性的物理靠山——AOD 移动光阱时，只要移动加速度不太大，原子就被"困在阱里跟着走"，不会因为运动而丢失量子信息。

ZAP 默认的原子搬运时间（15 μs per transfer）假设了"慢到足够绝热"。如果搬运加速 → trap depth 变浅 → 原子可能被甩出去 → f_tr 不再是 0.999。**f_tr 是搬运速度和加速度的函数，不是常数**——这是 ZAP fidelity 模型的物理过度简化之一。

> [gloss/01-phenomena]

## 三、架构层：Zone 分区

### 3.1 为什么需要 Zone

不分区 → 所有原子共享同一物理空间 → 任何位置的里德堡激光都会影响所有原子 →**空闲原子被误照射（串扰 crosstalk）**。

分区 → 物理上隔离"需要纠缠的区域"和"不需要纠缠的区域"。存储区在里德堡激光覆盖范围之外 → 空”原子零串扰。

**这是 ZAP 的核心硬件前提**——不是 ZAP 发明的（Lukin 组先做了实验验证），但 ZAP 是第一个把这个架构的编译潜力挖到底的。

> [ZAP §II-A]

### 3.2 Zone 架构的物理布局

```
┌─────────────────────────────────┐
│  纠缠区 (Entanglement Zone)      │
│  SLM + 全局里德堡激光             │
│  成对光阱：对内 4 μm，对间 ≥ 6 μm  │
│  只有此区的原子能做双比特门        │
│  容量：~160 阱 (80 对)            │
│                                  │
│  ←── 10 μm 间距 (激光隔离) ──→    │
│                                  │
│  存储区 (Storage Zone)            │
│  SLM 静态光阱，间距 6 μm          │
│  原子在此做单比特门 + 等待下次调用  │
│  容量：~80 阱                     │
│          AOD 搬运通道 ↔            │
└─────────────────────────────────┘
```

### 3.3 Zone 架构的核心 trade-off

- **纠缠区太小** → slot（可用双比特门站位）不够 → 频繁搬进搬出 → 搬运开销大
- **纠缠区太大** → 更多原子同时暴露在里德堡激光下 → 串扰风险上升 + 对间距必须加大 → 并行度反而下降

ZAP 当前 zone 尺寸是固定的（80 + 160 阱）。不同电路结构（QFT 的蝶形 vs QRAM 的树形）可能需要不同的最优 zone 配置。这是 zone sizing 作为编译参数的优化方向。

## 四、编译层：编译器做什么

### 4.1 量子编译器在技术栈中的位置

```
算法层 (VQE/QAOA/Shor...)  →  经典程序员写的数学
        ↓
量子高级语言 (Qiskit/Cirq) →  量子电路 = 门序列
        ↓
编译器 (ZAP/Enola/ZAC)     →  电路 → 物理原子调度    ← 我们在这里
        ↓
脉冲层 (OpenPulse/QUA)    →  物理指令 = 激光/微波脉冲序列
        ↓
测控硬件 (AWG/FPGA)       →  纳秒级物理信号打在原子上
```

编译器是软硬件之间的翻译官。它决定了：哪个逻辑比特映射到哪个物理原子、原子放在哪个坑位、双比特门用哪对纠缠坑、空闲比特留下还是搬走、搬运走哪条路线。这些决策直接进入 fidelity 公式，影响最终保真度。

> [eng/04-toolchain]

### 4.2 中性原子编译器的特殊之处

中性原子编译器和其他路线编译器的**本质区别**：原子可以搬。

| | 超导编译器 | 中性原子编译器 |
|---|---|---|
| 比特移动性 | 固定（焊死在芯片上） | 可移动（光镊搬运） |
| 纠缠条件 | 必须在耦合器连接范围内 | 搬到纠缠区即可 |
| 主要编译任务 | SWAP 路由（用门链间接连接） | 搬运调度（直接搬原子） |
| 串扰源 | 固定近邻串扰 | 里德堡激光全局照射 |
| 核心 trade-off | SWAP 数量 vs 电路深度 | 搬运次数 vs 串扰暴露 |

中性原子的可移动性是双刃剑：**搬得动 → 不需要 SWAP 链 → 电路更短；但搬需要时间 → 所有原子在搬的时候都在退相干 → 搬太多会丢原子。**

> [eng/04-toolchain], [ZAP §I]

### 4.3 ZAP 的编译策略：单遍确定性

ZAP 的核心哲学：**不迭代、不全局搜索、一遍过。**

传统编译器（如 ZAC）用模拟退火迭代优化初始布局 + 全局搜索最优搬运方案 → 编译质量可能更高 → 但编译慢 1000×+。ZAP 的选择是：用 routing-aware 的贪心 look-ahead 替代全局搜索。代价是可能不是全局最优，收益是编译速度碾压。

ZAP 编译流水线的四步：
1. **调度**：把 CZ 门按 ASAP 排期，单比特门填入剩余槽位（ASAP-separate）
2. **初始布局**：重要比特优先分配离纠缠区近的存储坑位，考虑 AOD 冲突
3. **逐 stage 布局**：每 stage 决定 keep-vs-move + 分配纠缠坑对
4. **路由**：有冲突时 parking（短暂位移错开身位），贪心最大独立集并行搬运

> [ZAP §III]

### 4.4 Fidelity 模型：编译器的计分板

ZAP 用 Eq.4 作为所有编译决策的统一度量：

```
F_total = (f_1)^g1 × (f_2)^g2 × (f_xtalk)^N_xtalk × (f_tr)^N_tr × Π exp(-t_q/T2)
          └─硬件决定─┘   └──────────── 编译器可优化 ────────────┘
```

五项分别对应：
1. **单比特门**（f_1 ~ 0.9997）——硬件参数，编译器几乎不影响
2. **双比特门**（f_2 ~ 0.995）——硬件参数，编译器决定做多少个
3. **串扰**（f_xtalk）——空闲原子被里德堡激光误照射 → keep-vs-move 决策直接影响
4. **搬运**（f_tr ~ 0.999）——原子 AOD↔SLM 转乘 → placement + routing 影响
5. **退相干**（T2 ~ 1.5×10⁶ μs）——空闲时间越长掉越多 → 调度影响

**前两项是固定税（硬件决定），后三项是编译器可以优化的。** ZAP 的 6 个决策点各自影响后三项的不同组合。

> [ZAP Eq.4, §II-B]

## 五、ZAP 的六个决策点

### 5.1 决策点全景

| # | 桥 | 问题 | ZAP 默认 | 我在做什么 |
|---|---|---|---|---|
| 1 | keep-vs-move | 空闲比特留下还是搬回？ | 硬阈值 Eq.15 | AL 联合优化 |
| 2 | λ_par | 并行度 vs 搬运距离？ | λ=1000 固定 | 电路自适应 |
| 3 | parking | 搬回存储区放哪个位置？ | 1 site | 多 site + trap 质量 |
| 4 | ASAP | 调度策略？ | separate | 电路驱动混合 |
| 5 | qubit priority | 哪些比特优先占纠缠区？ | 1/(l+1) | trap + state + reuse |
| 6 | idle α | 闲置代价换算系数？ | α=1.0 | 硬件自适应 |

**核心发现**：5/6 的桥在 TQE benchmark 尺度（10–30 比特）上已达 fidelity 平台期——ZAP 默认参数已近最优。只有 keep-vs-move 存在结构性优化空间。

> [ZAP §III, §IV], [本项目六桥基准]

### 5.2 最关键的桥：keep-vs-move

```
物理场景：
  纠缠区有 N 个空闲原子（刚做完门、下一 stage 不做门）
  纠缠区只有 K 个 slot 槽位（K 对光阱）

  STAY → 留在纠缠区 → 省搬运 → 但下一 stage 里德堡激光照过来会串扰
  MOVE → 搬回存储区 → 安全（零串扰）→ 但搬运花时间+可能的原子丢失

  Eq.15: if L_xtalk > L_tr + L_dec → MOVE; else → STAY
  (L_xtalk = 串扰代价, L_tr = 搬运代价, L_dec = 搬运期间的退相干)

问题：
  每个比特独立算账，不管全局 slot 容量
  当 K 很小（紧 slot），多个比特同时决定 STAY → 超过 K → slot violation
  硬阈值在紧 slot 下系统性高估可用容量

我的 AL 改进：
  连续权重 w∈[0,1] + 全局约束 Σ(1-w_i) ≤ K → 联合优化
  紧 slot 下 violation 168→0，常规场景 fidelity 无差异
```

> [ZAP §IV-C, Eq.15], [本项目 strategy_compare + tight_slot]

## 六、度量层：怎么评价编译器

### 6.1 核心指标

| 指标 | 含义 | ZAP 默认值 |
|---|---|---|
| **Fidelity** | 编译后量子态的正确概率 | Eq.4 五项乘积 |
| **编译时间** | 从电路到指令的时间 | 毫秒级（单遍） |
| **执行时间** | 编译后电路运行的物理时间 | 微秒级 |
| **Crosstalk** | 空闲原子被误照射的次数 × 串扰损失 | zone 架构 → ~0 |
| **Transfers** | 原子搬运次数 | 电路相关 |
| **Slot Violations** | 纠缠区超载次数 | 硬阈值有多有少 |

### 6.2 Fidelity Breakdown 速查

来自 ZAP 默认参数、TQE benchmark 的真实数据：

| Benchmark | 1Q Gate | 2Q Gate | Crosstalk | Transfer | Decoherence |
|---|---|---|---|---|---|
| QFT (10q) | 0.949 | 0.637 | **1.000** | 0.868 | 0.959 |
| Ising (26q) | 0.970 | 0.778 | **1.000** | 0.920 | 0.947 |
| GHZ (30q) | 0.983 | 0.865 | **1.000** | 0.890 | 0.850 |
| QRAM (20q) | 0.938 | 0.600 | **0.946** | 0.802 | 0.839 |
| Multiplier (15q) | 0.911 | 0.329 | **0.742** | 0.512 | 0.724 |

**关键观察**：结构化电路（QFT/Ising/GHZ）上 zone 架构保持 crosstalk=1.0——物理隔离完美。非均匀重用电路（QRAM/Multiplier）出现串扰衰减——keep-vs-move 的优化空间就在这些电路上。

## 七、扩展：相关的姊妹架构

### 7.1 QCCD（离子阱的分区架构）

离子阱路线的"传送带"方案：把芯片划成存储区、操作区、读出区，用电极电压搬运离子。哪两颗离子要做门，就运到操作区凑一对，做完送回去。

和 ZAP 的 zone 架构概念上同构——都是"分区+搬运"。但离子比原子更重 → 搬运更慢 → 逻辑时钟更慢。Quantinuum H 系列的量子体积纪录建立在它上面。

### 7.2 异构协同的未来

"超导快手 + 中性原子大规模 + 光量子总线"——三条路线各取所长。中性原子负责大规模存储和并行门操作，超导负责快速处理，光子负责远距离传输。编译器在这种架构下的新挑战：跨硬件类型的 unified IR + 任务拆分。

| **里德堡阻塞 (Rydberg blockade)** | 一个原子被激发成里德堡巨人 → 半径内邻居无法被同时激发 |
| **阻塞半径** | ~4.3 μm（n~70 的 Rb 原子），内部不可同时激发，外部互不理睬 |
| **范德华力 (van der Waals)** | 里德堡阻塞的力学来源：C₆/R⁶，对主量子数 n 极端敏感（C₆ ∝ n¹¹） |
| **退相干 (Decoherence)** | 量子态被环境噪声破坏——量子计算的头号敌人 |
| **T2 (相干时间)** | 量子相位信息保持的平均时间，ZAP 默认 1.5×10⁶ μs |
| **Zone 架构** | 存储区（候场）+ 纠缠区（手术室），物理隔离串扰 |
| **ASAP-separate** | ZAP 的调度策略：先排双比特门，再填单比特门 |
| **keep-vs-move** | 空闲比特留在纠缠区（串扰风险）还是搬回存储区（搬运代价） |
| **parking** | AOD 路线冲突时的短暂横向位移——错开身位后并行搬运 |
| **Fidelity (保真度)** | 量子态正确的概率——Eq.4 五项乘积 |
| **Slot** | 纠缠区中一对可用的双比特门站位 |
